# Hito 5: Despliegue de la aplicación en un IaaS o PaaS

## Descripción

Hacer un despliegue en la nube del proyecto de prácticas desarrollado. Además, de monitorizar el despliegue en tiempo real para detectar anomalías, optimizar el desempeño y garantizar resiliencia.


## Explicación

El principal objetivo de este hito es familiarizarse con las técnicas usadas para desplegar aplicaciones desde un repositorio web a un IaaS o PaaS (o usando ambos). Se debe automatizar el despliegue de la aplicación al IaaS (o PaaS) e incorporar herramientas de observabilidad.

Para ello debe seleccionar y darse de alta en el IaaS (o PaaS) gratuito (o de pago pero gratuito durante un tiempo o para un nivel determinado servicio) más adecuado para las necesidades de la aplicación que se va a desplegar. 

El IaaS (o PaaS) seleccionado tiene que cumplir tres requisitos:

* La configuración debe definirse en un fichero que describa la infraestructura. Se puede hacer o bien usando algún lenguaje o fichero de configuración que provea el IaaS (o PaaS), o bien mediante una secuencia de comandos introducidos a través de la herramienta de línea de órdenes (*toolbelt*) proporcionada por el IaaS (o PaaS). En ambos casos se tendrá que especificar claramente en la documentación de proyecto los pasos seguidos. El objetivo de esta parte es que una persona que se dé de alta y esté autorizada a usar ese IaaS (o PaaS) pueda, usando ese fichero de configuración o secuencia de comandos, reproducir la infraestructura y desplegar el mismo proyecto. Si el IaaS (o PaaS) sólo permite configuración desde la web, no sería válido, ya que tampoco se podría usar una aplicación tipo Terraform para aprovisionar la infraestructura.
* El IaaS (o PaaS) debe permitir el despliegue de la aplicación directamente desde el repositorio de GitHub. Es decir, que se pueda desde GitHub hacer el despliegue simplemente haciendo `push` desde el repositorio. La configuración de este despliegue se tendrá que documentar en README del repositorio de prácticas.
* Permitir el despliegue de la aplicación en Europa (legalmente obligatorio).

Debe abrir una cuenta en el proveedor de IaaS (o PaaS), crear la aplicación en la nube y ponerla en marcha. La herramienta que se use en el IaaS (o PaaS) para lanzar la aplicación tendrá que hacer uso de las órdenes que se hayan utilizado en los hitos anteriores. De esta forma se asegura que se ejecuta de la misma forma como se hacía localmente.

Además, de realizar el despliegue de la aplicación, ésta debe monitorizarse en tiempo real para detectar anomalías, optimizar el desempeño y garantizar resiliencia. Para ello debe incorporar herramientas de observabilidad para capturar métricas de desempeño, eventos, logs y trazas. Estas herramientas permiten reducir el tiempo de detección de fallos en la aplicación desplegada en la nube. Una vez implementadas estas herramientas, debe realizar una serie de pruebas para analizar las prestaciones de la aplicación desplegada en el IaaS (o PaaS), por ejemplo realizando pruebas de estrés para medir la carga o la latencia.



## Entrega de la práctica

Subir los fuentes a GitHub y hacer un *pull request* al fichero correspondiente en el repositorio de la asignatura.

Todos los elementos que se describen la rúbrica de valoración del hito tienen que estar correctamente identificados y enlazados desde el `README`, que reflejará el estado final del proyecto del estudiante. 

Se tendrá que incluir la URL donde se haya desplegado la aplicación en la nube.

## Valoración

- 1.5 puntos: Descripción y justificación de los criterios usados para elegir el IaaS (o PaaS) y las diferentes opciones valoradas.  
- 1.5 puntos: Descripción y justificación de las herramientas usadas para desplegar la aplicación en el IaaS(o PaaS).
- 1.5 puntos: Descripción de la configuración para el despliegue automático de la aplicación al IaaS (o PaaS) desde el repositorio de Github.
- 1.5 puntos: Descripción, justificación y configuración de las herramientas de observabilidad implementadas para monitorizar la aplicación desplegada en tiempo real.
- 1 punto: Funcionamiento correcto del despliegue en el IaaS(o PaaS) (no sólo el status, sino que correcto funcionamiento de la aplicación).
- 2 puntos: Pruebas de las prestaciones de la aplicación desplegada en el IaaS (o PaaS).
